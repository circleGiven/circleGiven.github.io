Javascript | 문장

2018-02-10 12:32:48
Dev>Javascript

자바스크립트 문장은 세미콜론(;)으로 끝난다.
표현식(Expression)과 다른점은 표현식은 어떤 값을 생성하기 위해 평가(evaluated)되지만,
문장은 어떤 일을 하기위해 실행(excuted)되는 것이다.

할당문이나 함수호출처럼 부수 효과가 있는 표현식은 독립적인 문장으로 사용할 수 있는데, 그런 것을 표현문(expression statement)이라 한다.
새 변수나 함수를 선언하거나 정의할때 사용하는 선언문(declaration statement)도 있다.

자바스크립트는 단순히, 순서대로 실행되는 문장들의 집합이다.
자바스크립트 인터프리터는 무장들을 순서대로 실행한다.
‘무언가 정말로 일어나게’ 만드는 또 다른방법은, 이 기본순서를 바꾸는 것이다.
이것은 자바스크립트가 제어 구조(control structure)가 존재하는 것을 의미한다.

  • if와 switch 같은 조건문(conditional)은 자바스크립트 인터프리터가 표현식의 값에 따라서 다른 문장의 실행 여부를 결정할 수 있게한다.
  • while과 for 같은 순환문(loop)은 특정 문장들을 반복 실행하는 문장이다.
  • break, return, throw 같은 도약문(jump)은 인터프리터가 프로그램의 특정부분으로 건너뛸 수 있게 해주는 문장이다.




표현문

자바스크립트 문장에서 가장 간단한 형태는 부수 효과가 있는 표현문이다.
할당문(assignment statement)이 바로 이러한 표현문 중 하나이다.

++나 --같은 증감 연산자들은 할당문과 관계가 있다.
이들은 변수 값 자체를 바꾸는 부수 효과가 있다.

delete연산자는 객체 프로퍼티를 삭제하는 부수 효과가 있다.
따라서 표현식의 일부로 쓰이기보다는 문장형태로 많이 쓰인다.

표현문의 다른 종류로 함수 호출이 있다.

위 alert과 close 함수 호출은 표현식 이지만, 웹브라우저에 영향을 주는 부수 효과가 있다.
부수 효과가 없다면 함수 표현식의 일부거나 할당문의 일부가 아닌 이상 호출하는 의미가 없다.
따라서 아래와 같이 할당을 해서 된다한다.




#복합문과 빈 문장
하나의 표현식 안에 여러 표현식을 합칠 때에는 쉼표(,) 연산자를 사용한다.
유사하게, 문장 블록(statement block)은 여러 문장을 하나의 복합문(compound statement)으로 묶는다.
여러문장을 중괄호({})로 감싸면 된다.

문장 블록에 대해 주의사항이 있다.

  • 문장 블록은 세미콜론(;)으로 끝나지 않는다.
    블록 내부의 기본문장은 세미콜론으로 끝나지만, 블록 자체는 끝나지 않는다.
  • 블록 안에 있는 구문은 구문들을 감싸는 중괄호({}) 위치를 기준으로 들여쓰기를 해야한다.
    필수는 아니지만 코드의 가독성과 이해도를 높인다.
  • 블록 단위의 유효 범위(scope)가 존재하지 않기 때문에, 구문 블록안에 선언된 블록뿐 아니라 블록 밖에서도 접근할 수 있다.

표현식이 다른 표현식을 포함하는 것처럼, 많은 자바스크립트 문장은 다수의 하위문(substatement)을 포함한다.
예로 while 루프 문법은 루프 몸체로 하나의 문장을 허용한다.
문장 블록을 사용하면 문법이 허용하는 하나의 하위문 자리에 많은 문장을 넣을 수 있다.

복합문은 하나의 문장이 있어야 할 곳에 여러 문장을 사용할 수 있다.
반대로 빈 문장은 하나의 문장이 있어야 할 곳에 아무런 문장도 두지 않을 수 있도록 한다.

자바스크립트 인터프리터는 빈 문장을 만나면 아무것도 실행하지 않는다.

선언문

var와 function은 각각 변수와 함수를 선언하고 정의하는 선언문이다.
이러한 선언문은 각각 변수나 함수 이름 같은 식별자를 정의한다.
선언문에서 정의된 식별자는 프로그램 내에서 사용할 수 있고, 할당문의 좌변 값으로 사용된다.

- var

var문은 하나 또는 그 이상의 변수를 선언한다.

var키워드 다음에는 선언될 변수들이 쉼표로 구분되어 이어진다.
선언만 할 수도 있으나, 초기 값을 지정하기 위해 표현식(initializer)을 가질 수도 있다.

함수 내부에서 사용하는 var문은 지역 변수를 정의하며,
그 지역 변수는 해당 함수 내에서만 사용할 수 있다.

만일 var문이 최상위 코드로 사용된다면, 이는 전역 변수를 선언하는 것이다.
전역 변수는 전역 객체의 프로퍼티 이나, 다른 전역 프로퍼티와는 달리 var로 선언된 프로퍼티는 delete로 삭제할 수 없다.

var문에서 변수에 초기값을 지정하지 않으면 변수의 초기값은 undefined가 된다.

var문은 for문 내지는 for/in루프의 일부로 사용할 수도 있다.
(이들 변수는 최상단으로 끌어올려진다)

같은 변수를 여러번 선언해도 괜찮다.
다만 이전의 값은 사라지게 된다.


- function

function키워드는 함수를 정의하는데 쓰인다.
함수 정의 표현식도 문장 형태로 사용할 수 있다.

함수 선언문의 문법은 다음과 같다.

‘함수이름’은 선언되는 함수의 이름이 될 식별자이다.
함수 이름 다음에 괄호 속 쉼표(,)로 구분된 전달인자의 목록이 온다.
함수가 호출되면 함수 몸체 내부에서는 이 식별자(전달인자)들을 사용하여 전달인자의 값을 참조하게 된다.

함수의 몸체는 하나 혹은 여러개의 자바스크립트 문장들로 구성되며, 문장들은 중괄호 안에 담겨있다.
이 문장들은 함수가 정의될 때는 실행되지 않는다.
대신, 함수가 호출되면 실행될 수 있도록, 새로운 함수 객체와 연결된다.

function문에는 중괄호({})가 반드시 필요하다.
for문 등에서 사용되는 문장 블록고는 다르게, 함수 몸체에는 하나의 문장만 오더라도 중괄호가 반드시 필요하다.

함수 선언문은 자바스크립트 최상위 단계 코드에서 나타날 수도 있고, 다른 함수 내에 중첩될 수도 있다.

하지만 함수가 다르 함수 속에 중첩될 때는, 중첩된 함수 내에서 최상위 단계에 위치해야 한다.
(함수 선언은 if나 while문 등의 다른 문장 안에 있을 수 없다.)
이와 같은 위치 제약 때문에, ECMAScript 명세는 함수 선언을 진정한 문장으로 분류하지 않는다.

함수 선언문은 함수 이름을 포함한다는 점에서 함수 정의 표현식과는 차이가 있다.
둘 다 새 함수 객체를 만들지만, 함수 선언문은 함수 이름을 변수로 선언한 후 이 변수에 함수 객체를 할당한다.
var로 선언한 변수와 같이, 함수 선언문으로 정의된 함수는 함수 유효범위의 최상단에 위치하게 된다.

var문을 이용한 함수 정의 표현식의 경우 변수의 선언만 유효범위 최상단으로 끌어올려진다.
(변수 초기화 코드는 원래 위치에 그대로 유지)

함수 선언문을 이용하면 함수의 이름과 본문 모두 유효범위 최상단으로 끌어올려진다.
이떄 스크립트 내의 모든 함수 또는 함수의 중첩 함수는 다른 코드가 실행되기 전에 선언된다.
이는 자바스크립트 함수를 호출하는 코드가 선언문이 나오기 전에도 올 수 있다는 뜻이다.
var문처럼 함수 선언문은 삭제할 수 없는 변수를 만든다.
(하지만 읽기전용 변수가 아니므로 값은 바뀔 수 있다.)



조건문

조건문은 특정 표현식의 값에 따라 문장을 실행하거나 건너뛴다.
이러한 문장은 분기라고도 하며, 일종의 결정 포인트이다.

- if

if문은 자바스크립트에서 단순히 어떤 결정을 내리거나,
조건에 따라 좀 더 정교하게 문장들을 실행할 수 있도록 하는 가장 기본이 되는 제어문이다.

‘표현식’의 값이 true(truthy)이면 ‘문장’이 실행된다.
(false(falsy)이면 실행되지 않는다)

if문에서 여러 문장을 쓰려면 반드시 괄호가 와야한다.

if문은 else절을 포함 할 수도 있다.
이else는 if문이 false가 되면 실행된다.

표현식이 false(fasly)라면 ‘문장2’가 실행된다.

else문 역시 여러 문장을 쓰도록 괄호를 이용할 수 있다.

중첩된 if문에서 else를 사용할 때 주의사항이 있다.

이 예제는 안쪽의 if문은 바깥쪽 if문에 중첩되어 있다.
그리고 사용자는 else를 바깥쪽 if문이 false일 경우 작동하도록 의도하였다.

그러므로 else문을 사용할 때는 중괄호를 사용해야한다.

- else if

if/else문은 조건문을 평가하고 결과에 따라 둘 중 한 코드를 실행한다.
하지만 분기가 두 개가 아니라 여러 개라면?
이때 사용하는것이 바로 else if이다.

- switch

if문은 동일한 표현식이 반복되는 경우 무의미하다.
switch문은 if문에서처럼 동일한 표현식이 여러번 반복되는 문제점을 해결한다.

switch문은 아래와 같은 작동 방식에 따른다.

  • switch문이 실행되면 표현식의 값을 계산하고 이 값에 해당하는 case레이블을 찾는다.
    (값이 같은지 판단하기 위해 ===연산자가 사용된다)
  • 만약 대응하는 레이블을 찾으면 해당 case레이블 다음에 오는 문장을 순서대로 실행한다.
  • 표현식과 대응하는 레이블을 찾지 못한 경우 default: 레이블을 찾는다.
  • default:레이블을 찾았을 경우 default:레이블 다음에 오는 문장을 순서대로 실행한다.
  • default:레이블을 찾지 못했을 경우 모든 코드블록을 건너 뛰고 switch문을 종료한다.
