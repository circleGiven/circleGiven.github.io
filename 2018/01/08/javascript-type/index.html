숫자 Number

자바스크립트는 다른언어와 달리 정수와 실수를 구분하지 않는다.
자바스크립트는 모든 숫자를 실수로 표현하는데 IEEE 754 표준에서 정의한 64비트 실수 형태로 표현한다.
하지만, 배열 인덱싱이나 비트 연산같은 특정연산은 32비트 정수로 수행한다.
그리고 숫자는 표현할 수 있는 특정 범위를 갖는다. (Number 객체를 이용하면 알 수 있다.)


- 리터럴 numeric literal

프로그램 안에 있는 그대로 등장하는 숫자
자바스크립트에서는 여러 숫자 리터럴을 지원한다.

> 정수 리터럴

자바스크립트는 10진수와 16진수를 지원한다.

하지만, ECMAScript 표준에서는 8진수를 지원하지 않는다.
그도 그럴것이 사용자가 한눈에 보기에 8진수와 10진수 인지 구분할수 없다.


> 부동소수점 리터럴

실수는 다음과 같이 표현 할 수 있다.

지수표기법으로도 표기가 가능하다.
e23의 경우 10^23 을 의미한다
E-32의 경우 10^-32를 의미한다.

- 산술 연산

자바스크립트에서는 0을 나누거나 0으로 나눌때 에러가 발생하지 않는다.

> 오버플로 overflow

표현할 수 있는 가장 큰 수(Number.MAX_VALUE)보다 크다면 Infinity라고 표시한다.

반대로 표현할 수 있는 가장 작은 음수(-Number.MAX_VALUE)보다 작다면 -Infinity가 된다.


> 언더플로 underflow

표현할 수 있는 0에가까운 값(Number.MIN_VALUE)보다 0에 가까울 경우 0이라고 표시한다.

반대로 표현할 수 있는 음수에서 0에가까운 값(-Number.MIN_VALUE)보다 0에 가까울 경우 -0이 된다.

그리고 0을 0으로 나눌때는 위와는 달리 독특한 값 NAN이 나온다.

이 NAN이라는 값은 숫자가 아닌 다른 타입으로 산술연산을 시도할때나,
무한대(Infinity)를 무한대로 나눌때 발생한다.

그런데 여기서 위 값들을 비교하면 특이한 현상이 발생한다.

상식적으로 NaN과 NaN을 비교하면 당연히 true가 나올거라고 생각할 것이다.
하지만 NaN은 그 자신뿐만아니라 다른 값과 같은지 비교를 할 수 없다.

그러므로 변수 x가 NaN인지 ===연산자를 사용할 수 없다는 것이다.
비교를 하려면 NaN자신은 NaN이 아니다라는 특징을 사용해야된다.


만약 자바스크립트에서 유효한 숫자인지 판단하는경우 아래와 같이 사용하면 된다.


- 이진 부동소수점과 반올림 오류

자바스크립트에선 한정된 숫자만 부동소수점 형태로 표현할 수 있다.
실수연산을 할 때는 근사 값으로 표현한다.
부동소수점 표현 방식(IEE-754)은 이진 표현법을 사용하고 있다.

그래서 아래와 같은 현상이 발생한다.

상식적으로 a는 0.1을 나타낼거라 생각한다.
하지만 이것은 이진 부동소수점 숫자를 이용하기 때문에 발생하는 것이다.
실제로 a를 찍어보면 0.09999999999999998 이라고 나온다.



텍스트 Text

문자열은 16비트 값들이 연속적으로 나열된 변경이 불가능한 값이고, 각 문자는 유니코드로 표현된다.
문자열의 길이는 문자열에 들어있는 16비트 값의 개수이다.

그러므로 사용자가 “a”라고 입력을 하여도 길이를 계산할때 자바스크립트 내부에서 유니코드로 변경후 계산한다.
이 특성때문에 다음과 같은 현상이 발생한다.

문자 𝑒는 사용자가 보기에 한 글자이지만, 유니코드상으로 표기하면 \ud835\udc52라는 값이다.
두 개의 유니코드 값으로 이루어져 있기 때문에 길이를 계산하면 2로 표시되는 것이다.

- 문자열 리터럴

자바스크립트에서 문자열을 표현하려면 큰따음표", 작은따음표', 백틱`으로 감싸면 된다.

만약 큰따음표로 감싼 문자열 내에서 큰따음표를 표현할때 이스케이프 문자열을 사용해야된다.

- 이스케이프 문자열

역슬래시(\)는 자바스크립트 문자열에서 이스케이프 하는데 사용된다.
큰따음표로 감싼 문자열 내에서 큰따음표를 표현할때 이것을 이용해야된다.


시퀸스 표현하는 문자
\0 (\u0000) null 문자
\b (\u0008) 역스페이스
\t (\u0009) 수평 탭
\n (\u000A) 줄바꿈 문자
\v (\u000B) 수직 탭
\f (\u000C) 폼 피드
\r (\u000D) 캐리지 리턴
\” (\u0022) 큰따음표
\’ (\u0027) 작은따음표
\ (\u005C) 역슬래시
\x 두 개의 16진수 숫자
\u 네 개의 16진수 숫자

이스케이프는 그 자체가 유니코드로 되어있기 때문에 길이를 구할때 하나로 취급된다.


- 문자열 다루기

자바스크립트에서 문자열을 이어 붙일때 +연산자를 사용한다.

그리고 문자열은 변경이 되지않는다.
예를들어 replace()와 toUpperCase()같은 메서드를 사용할 때 새 문자열을 반환한다.


- 패턴매칭

문자 패턴을 나타내는 객체를 생성하기 위해 RegExp() 생성자를 정의한다.
이것을 정규표현식이라 부르고, 펄(Perl)의 구문을 따른다.
그리고 자바스크립트 문자열은 RegExp객체를 인자로 갖는 메서드를 가지고 있다.




불리언 Boolean

불리언 값은 참/거짓을 표현한다.
일반적으로 비교의 결과로 생성한다.

변수 a가 값 4와 같은지 검사를 한다.
같다면 true를 반환하고, 다르다면 false를 반환한다.

> falsy 값

아래 값은 모두 false를 반환한다.

  • undefined
  • null
  • 0
  • -0
  • NaN
  • “” //빈 문자열

이 이외에 모든 값은 true를 반환한다.



null과 undefined

null은 아무 값을 갖지 않음을 나타낸다.

그러나 null은 정말 아무 값이 없는게 아닌 ‘아무 값이 없음’을 나타내는 고유의 자료형이다.

자바스크립트에는 값이 없음을 나타내는 다른값 undefined가 있다.
undefined는 null보다 더 엄격한 의미를 나타낸다.
초기화 되어있지 않은변수, 존재하지 않는 객체 프로퍼티에 접근 할때 나타나는 값이다.
둘다 falsy한 값으로 표현이 되지만 둘이 완전히 같은 값은 아니다.



전역 객체 global object

전역 객체는 자바스크립트 프로그램 전역에서 사용할 수 있게 정의한 심벌이다.
자바스크립트 인터프리터가 시작할때(웹브라우저가 새로운 페이지를 불러올 때), 새로운 전역객체를 만들고 프로퍼티들을 초기화한다.

  • undefined, Infinity, NaN //전역 프로퍼티
  • isNaN(), parseInt(), eval() //전역 함수
  • Data(), RegExp(), String(), Object(), Array() //생성자 함수
  • Math, JSON // 전역 객체

위 전역프로퍼티들은 예약어는 아니지만 예약어처럼 취급된다.
전역객체는 this라는 키워드로 참조할 수 있다.
전역객체가 최초 생성될때, 자바스크립트에서 사용하는 모든 전역 값이 정의된다.



래퍼 객체 Wrapper

객체는 프로퍼티 또는 이름있는 값들의 집합이며, .표기법을 이용하여 프로퍼티의 값을 참조한다.
프로퍼티 값이 함수라면, 그 값을 메서드라고 부른다.
객체 o의 메서드 m을 호출하려면, o.m()이라고 적으면된다.

그런데 좀 특이한 점이 있다.
문자열의 길이를 구할때 .length를 사용하였다.
왜 문자열은 객체가 아닌데 length프로퍼티를 가지고 있을까?

사용자가 문자열(x)의 프로퍼티를 참조하려고 할때, 자바스크립트는 new String(x)를 호출한것 처럼 객체로 반환한다.
이를 래퍼 객체라고 한다.

프로퍼티 참조를 하면 임시객체(래퍼객체)가 생성되는데, 프로퍼티 참조가 끝나면 임시객체는 메모리에서 회수된다.

a.len에 값 4를 할당했지만, 실제로 a는 임시객체였기 때문에 참조가 끝나면 메모리에서 삭제된다.

그래서 var b = a.len;를 할 시점에 값이 undefined가 된다.

이는 문자열 뿐만아니라 숫자,불리언도 같은 이유로 메서드를 가지고 있다.
역시 프로퍼티를 참조를 하면 Number(), Boolean() 임시객체를 생성하고 반환한다.

그러나 null과 undefined는 래퍼객체가 없다.
래퍼객체와 일반타입은 마치 같은것처럼 보이는 효과가 있다.

그러나 실제로 각자 값을 찍어보면 a는 일반타입이지만, b는 String객체라는 것이 나온다.
둘을 비교할때는 ===연산자를 사용해야된다.

그리고 typeof연산자를 통해 차이를 알 수 있다.




변경 불가능한 원시타입 값과 변경가능한 객체

자바스크립트에서 원시타입과 객체는 근본적인 차이점이 있다.
원시타입은 변경이 불가능하다.
숫자의 경우 바꾸면, 다른 수가 된다. 불리언도 마찬가지이다.

그러나 문자열의 경우 다르게 생각할 수 있다.
기본적으로 문자열이라는 것은 문자의 배열로 볼 수 있다.
배열이므로 배열의 원소를 변경하는것과 같이 문자열도 변경할 수 있다고 생각할 것이다.

하지만 자바스크립트에서는 각 문자를 변경할 수 없다.

그리고 원시타입은 값으로 비교된다.

변수에 할당할때도 값을 집어넣는다.


객체는 변경이 가능하다.

그리고 객체는 참조로 비교딘다.

객체는 참조타입(reference type)이라고 불린다.
객체는 참조를 할당한다.

- 변환과 동치

자바스크립트는 값의 타입을 유연하게 변환시킬 수 있는데,
동치연산자 ==도 같은 원리로 동작한다.

==연산자가 비교를 하기전에, 내부적으로 타입변환이 일어난다.
특이한것이 있는데, undefined의 경우 false로 변환은 일어난다.

그러나, undefined == false는 아니다.


- 암시적 변환

자바스크립트는 특정한 연산자를 사용할때 암시적으로 변환이 일어난다.


- 명시적 변환

자바스크립트는 자동적으로 타입변환을 수행하지만 사용자가 원할때 명시적으로 변환을 할 수 있다.
명시적 변환을 할때 가장 간단한 방법은 래퍼 객체를 사용하는 것이다.

자바스크립트는 숫자를 문자로 변환하거나 문자를 숫자로 변환 때, 결과 값을 자세히 표현할 수 있게 메서드를 지원한다.
Number 클래스의 toString()은 기수를 정하는 선택적 인자를 받는다.
인자가 없는경우 10진수로 표시된다.

그리고 소수점 이하의 자리수를 제어할때도 메서드를 지원한다.

Number() 객체는 명시적변환을 할때 숫자가 아닌 값은 NaN을 일으킨다,

그럴경우 parseInt()와 parseFloat()을 이용할 수 있다.
parseInt()는 정수만 변환할 수있지만,
parseFloat()는 정수와 부동소수점 모두 변환할 수 있다.

뒤에 숫자가 아닌 값이 올 경우에는 제거후 숫자로 인식이 되지만,
앞에 숫자로 인식되지 않는 값이 오면 NaN을 반환한다.

또한 parseInt와 parseFloat의 두번째 인자로 숫자 값을 전달하면, 두번째 인자를 기수로 첫번째 인자를 변환시킨다.


- 객체에서 원시타입으로 변환

모든 객체에서 불리언으로 변환은 항상 true로 반환된다.
객체에서 문자열로 변환될때, 숫자로 변환될때는 변환될 객체의 메서드를 호출함으로 수행된다.

모든 객체는 두 개의 타입변환 메서드 toString()과 valueOf()를 상속한다.
toString()는 객체를 문자열로 표현한다.
그리고 각 클래스마다 toString()을 재정의해서 서로 다르게 표시한다. (그래도 결국 문자열로 표시한다.)

valueOf()는 본래 객체를 잘 표현하는 원시타입 값을 반환해야한다.

그런데, 객체의 경우 결합된 값이다.
대부분 객체는 하나의 원시타입으로 표시할수 없기때문에 valueOf()는 객체 그 자신을 반환한다.



객체는 다음과 같은 절차로 문자열로 변환된다.

  • 객체가 toString() 메서드를 가지고 있는지 찾는다.
    찾았다면 toString()을 호출한다.
    원시타입 값을 반환한다.
    이 값이 문자열 이면 그대로 출력하고, 아니면 값을 문자열로 변환하여 반환한다.
  • toString()을 못찾았을 경우, valueOf() 메서드를 가지고 있는지 찾는다.
    찾았다면 valueOf()를 호출한다.
    원시타입 값을 반환한다.
    이 값이 문자열 이면 그대로 출력하고, 아니면 값을 문자열로 변환하여 반환한다.
  • 두 메서드를 못찾았을 경우, TypeError를 반환한다.



객체는 다음과 같은 절차로 숫자로 변환된다.

  • 객체가 valueOf() 메서드를 가지고 있는지 찾는다.
    찾았다면 valueOf()을 호출한다.
    원시타입 값을 반환한다.
    이 값이 숫자라면 그대로 출력하고, 아니면 숫자로 변환하여 반환한ㄷ.
  • valueOf()을 못찾았을 경우, toString() 메서드를 가지고 있는지 찾는다.
    찾았다면 toString()를 호출한다.
    이 값을 숫자로 변환하여 반환한다.
  • 이 외의 경우, TypeError를 반환한다.

그렇다면 이제 빈 배열이 숫자로 변환시 0으로 변환되는지 알 수 있다.
배열은 기본적으로 valueOf()가 원시타입을 반환하지 않고 객체를 반환한다.

그러면 객체에서 숫자로 변환에서 두번째 규칙을 따른다.
toString()메서드를 호출하는데 빈 문자열 ""가 반환될 것이다.
이 빈 문자열은 다시 숫자로 변환되어 0이 나오게 된다.



변수 선언 declaration

자바스크립트에서 변수 선언은 var 키워드를 사용한다.

하나의 키워드로 여러개의 변수 선언도 가능하다.

선언과 동시에 초기화도 가능하다.

만약, 초기화를 하지 않는다면, 변수에 값이 할당되기 전까지 undefined값을 갖는다.

자바스크립트에선 특이하게 엄격하게 타입구분이 없기때문에 다른언어와는 다른 현상을 보인다.

만약 선언되지 않은 변수를 호출 할 경우 에러가 발생한다.




변수의 유효범위

변수의 유효범위란 어떤 변수가 정의되어 있는 영역을 말한다.
전역(global)과 지역(local)으로 나누어 진다.
전역변수는 말그대로 모든 전역에서 유효하다.
지역변수는 유효범위가 한정적이다.

같은 이름의 지역변수와 지역변수가 선언되었을 경우, 함수 내부에서는 지역변수가 우선권을 갖는다.

전역변수에서는 var키워드를 안써도 전역변수로 선언이 되지만,
지역변수는 반드시 var를 써야된다.


- 함수 유효범위와 끌어올림 hoisting

다른 프로그래밍 언어에서 블록 안에 있는 코드는 자신만의 유효범위를 가지며,
블록 안에서 선언된 변수는 블록 밖에서 볼 수 없다.
이를 블록 유효범위 (block scope) 라고한다.

자바스크립트는 블록 유효범위의 개념이 없다.

대신에, 함수 유효범위(function scope)를 사용한다.
어떤 함수 안에서 선언된 모든 변수는 그 함수 전체에 걸쳐 유효하다.
변수가 정의된 함수 안에서 보일 뿐 아니라, 그 함수 안에 중첩된 함수 안에서도 보인다.

다음은 변수 i,j,k 모두 다른위치에 선언되지만 모두 같은 유효범위를 갖는다.

이런 특징은 끌어올림(hoisting) 때문이다.
호이스팅은 모든 변수선언만 끌어올리고, 초기화 까지는 끌어올리지 않는다.

맨 처음 console.log(scope);에서 undefined가 아닌 global을 예상할 것이다.

그러나, 호이스팅의 원리로 인해 함수가 선언될때 두번째 줄에 있는 var scope = "local";이 함수 가장위로 올라와 선언된다.
때문에 undefined가 나오는 것이다.

- 프로퍼티의 변수

전역 변수로 선언한다는 것은, 곧 전역 객체의 프로퍼티를 정의하는 것이다.
변수를 선언시 var를 사용하면, 생성된 프로퍼티는 수정이 불가능하고 delete연산자로 제거가 불가능하다.

그러나 선언하지 않고 값을 할당하면, 자바스크립트는 자동으로 전역변수를 만든다.
이 값은 수정이 가능하고 삭제도 가능하다.

this키워드는 전역 객체를 참조할 수 있다.

하지만, 지역변수를 참조하는 키워드는 없다.


- 유효범위 체인 scope chain

자바스크립트의 모든 코드 무더기는 유효범위 체인(scope chain)을 가지고 있다.
유효범위 체인은 범위 안에 있는 변수를 정의하는 객체의 체인, 리스트이다.

자바스크립트가 변수 x의 값을 얻을때, 처음 체인에서 x를 찾는다.
없는경우 체인에 있는 다음 객체에서 x를 찾는다.
이런식으로 x를 찾지 못할 경우 ReferenceError가 발생하게 된다.

> 최상위 자바스크립트에서 유효범위 체인

어떤 함수에도 속하지 않는 코드에서, 유효범위 체인은 단 하나의 전역 객체만으로 이루어진다.

> 중첩되지 않은 함수의 유효범위 체인

중첩되지 않은 함수(함수내에 함수가 없는경우)에서, 유효범위 체인은 두 개로 이루어진다.
함수 매개변수와 지역 변수를 정의하는 객체와, 전역 객체로 이루어진다.

> 중첩된 함수의 유효범위 체인

중첩된 함수(함수내에 함수가 있는경우)에서, 유효범위 체인은 세 개로 이루어진다.

함수 매개변수와 지역 변수를 정의하는 객체와, 전역 객체로 이루어진다.
중첩함수가 호출될 때, 해당함수의 지역변수를 저장하기 위해서 새로운 객체를 하나 생성하고, 해당 객체를 기존에 저장된 유효범위 체인에 추가한다.
(중첩함수의 경우, 함수를 호출할 때마다, 중첩된 함수(내부 함수)가 매번 선언된다. 그래서 호출할 때 매번 유효범위 체인이 달라진다.)
