버전 관리 시스템 이란?

여기 두명의 개발자가 있다.

개발자 A : XXXX메서드 새로 추가했어
개발자 B : 어 그럼 난 AAAA메서드를 추가할께
개발자 A : 알겠어.

이처럼 변경이 일어난 한 파일에 대해 어떤 식으로 관리를 할까?
그럴때 필요한 것이 버전 관리 시스템(Version Control System; VCS)이다.
버전 관리는 파일이 시간을 기점으로 어떤 사람이, 어떤 변화가 일어났는지를 기록한다.
새로 추가하는 것 뿐만 아니라 다음과 같은 상황에서도 사용한다.

개발자 A : EE 파일 변경했어.
개발자 B : 어 알겠어. 근데 AAAA메서드 어디갔어? 이거 변경하면서 지운것같은데?
개발자 A : 앗! 아아..
개발자 B : 어쩔수 없지. 원복할테니 파일 다시 수정해줘.

이런 경우 마지막으로 파일 변경이 일어나기 전으로 돌려서 파일을 원상복구 시킬수 있다.

버전관리는 세가지가 있다

  • 로컬 버전관리 시스템
  • 중앙집중식 버전관리 시스템
  • 분산 버전관리 시스템

분산 버전관리 시스템 중 하나가 Git 이다.

Git

Git은 Subversion과 달리 스냅샷 방식으로 저장하고 관리한다.
