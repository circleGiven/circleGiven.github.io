Typescript | Types

2018-01-17 16:59
Javascript>Typescript

해당 글은 https://www.typescriptlang.org 을 참조하여 작성하였습니다.

타입스크립트에는 아래와 같은 타입들이 있다.
타입스크립트는 자바스크립트로 만들어져 있기때문에 자바스크립트의 타입을 모두 가지고 있다.
그리고, 타입스크립트에서 선언한 타입과 다른 타입의 값을 넣으면 컴파일 과정에서 에러가 발생한다.

number

정수, 소수 이외에 2진수나 8진수 등 모든 숫자가 가능하다.



boolean

오직 true나 false 값을 갖는다.



string

문자값을 가지며 ', ", ` 모두 사용 가능하다.

특히 backtick/backquote(`)는 위 두개와 달리 이런식으로도 응용이 가능하다.

뿐만아니라 ${} 안에서 연산도 가능하다.



null

의미는 자바스크립트에서의 의미와 같다.
선언이 되었고, null 이란 값을 할당한 상태이다.
사실상 이 타입은 타입선언으로선 거의 쓸 일이 없다.



undefined

의미는 자바스크립트에서의 의미와 같다.
선언은 되었지만 아무런 값을 할당하지 않은 상태이다.
사실상 이 타입은 타입선언으로선 거의 쓸 일이 없다.



object

자바스크립트에서와 같이 객체 및 배열 둘다 가능하다.



아래부터는 자바스크립트에 없고 타입스크립트에 새로 추가된 타입들이다.

Array

오로지 배열만 가능한 타입이다.

배열은 두가지 방법으로 사용이 가능하다.

타입을 선언하고 뒤에 []를 붙이는 방법과

generic으로 선언하는 방법이 있다.



Tuple

자바스크립트에는 없는 타입이다.
기존 자바스크립트에서 배열내에 자유로운 타입이 올 수 있었다.

예를들어

이런식으로 number형과 string형 처럼 다른타입이 같은 배열에 있어도 에러가 발생하지 않았다.

그러나 Tuple을 사용하면 에러가 발생한다.

초기화 할때를 제외하고는 3번째 index 부터는 number 나 string값 모두 가능하다. (이때부턴 union으로 적용된다.)

사실 왜 있는지 모르겠다.


Enum

흔히 다른 언어에 있는 Enum타입이다.
데이터 유형을 만들때 편리하다.

그리고 다음과 같이 초기화도 가능하다.



any

어떤 타입이 올지 예상 할 수 없는경우, 컴파일 에러를 피하기위해 많이 사용한다.

any와 object와 비슷하다 생각 할 수 있는데, 차이가 있다.

실제로 컴파일 과정에서는 통과하지만 런타임의 경우 에러가 발생 할 수 있다.
(런타임 과정에서는 number로 정의되기 때문)

그러나 object의 경우 컴파일에서 바로 체크가 가능하다.

any는 배열에서 아래와 같은 방식으로 사용이가능하다.
그리고 내부에는 어떤 타입이든 올 수 있다.



Void

any형과는 반대로 어떤 타입도 가지지 않는다.
일반적으로는 함수에서 반환형으로 가능하다.

물론 변수도 선언이 가능하긴 하다.

그러나 undefined와 null 이외에 어떤 값도 올 수 없기때문에 실제로 사용되지 않는다.


Never

어떠한 값도 올 수 없다한다.
심지어 undefined도 올 수 없다.

문서에는 반환 값이 없거나 항상 예외를 던지는 거라고 한다.

솔직히 왜 있는 타입인지 알 수 없다.
아직까지 사용해 본 적이 없다.
