Ajax | Ajax란?

2018-04-03 22:57:36
Ajax

Ajax의 정의

전통적인 웹이 편지를 쓰는 것이라면 Ajax는 메신저 대화를 하는 것이다.
- 데렉 포와젝

전통적인 웹사이트의 경우 브라우저는 서버에 전체 페이지를 요청한다.
그러고 나서 사용자가 링크를 클릭하거나 폼을 전송하면 그 시점에서 브라우저는 서버에 새로운 요청을 보낸다.
그 다음 서버는 새로운 다른 페이지를 보낸다.

클라이언트가 서버로 요청을 보내고 서버는 페이지 전체를 응답으로 보낸다. 모든 페이지를 갱신하고 이 작업을 반복한다.

Ajax 방법론은 페이지 기반 모델에서 벗어난 것이다.
사용자가 링크를 클릭하거나 폼을 전송하는 등의 방법으로 페이지와 상호작용을 할 때, 서버는 분리된 정보 조작을 반환한다.
전체 페이지를 반환하는 대신 현재 로딩되어 있는 페이지의 일부만 변경된다.

데이터가 클라이언트와 서버 사이에서 조각으로 나뉘어져 전송된다.
서버는 전체 페이지를 반환하지 않는다.

전통적인 웹사이트가 시작-정지의 행동성향을 제공했다면,
Ajax 어플리케이션은 더 빠르고 응답성이 높은 반응을 끊임없이 제공한다.

전체 페이지를 새로 고치지 않고 서버와 통신하는 방법, Ajax



웹 페이지의 일부를 서버에서 전송받은 정보로 갱신하는 것은 멋진 효과이다.

XMLHttpRequest

XMLHttpRequest 객체는 자바스크립트를 사용해 웹 페이지에서 서버와 통신할 수 있도록 해주는 자바스크립트 확장 기능이다.
Ajax는 대게 XMLHttpRequest 객체를 사용한다는 것과 같은 의미다.
(Ajax 개발자도 XMLHttpRequest를 생각해 만들었다)

Ajax 툴킷

XMLHttpRequest 객체는 Ajax를 움직이는 엔진이지만 XMLHttpRequest 객체만으로 Ajax를 사용할 수는 없다.

Ajax는 한 가지 기술이 아니다. 독립적으로 널리 사용되는 여러가지 기술이 새롭고 강력한 방식으로 결합된 것이다.
Ajax는 방법론이다.
- 제시 제임스 가렛


서버 측 언어

개발자들은 클라이언트의 입력에 따라 지능적으로 어플리케이션이 응답하기를 원한다.
서버에서는 브라우저로 보내야 할 정보가 무엇인지 결정할 수 있어야한다.
이러한 작업을 수행하기 위해서 서버 측 프로그래밍 언어를 사용할 필요가 있다.

- XML?

XML이 Ajax 어플리케이션의 결정적인 요소라 생각할 수도 있다.
Ajax의 주요 요소인 XMLHttpRequest라는 용어가 잘못된 인식을 심어주고 있기 때문이다.

확장성 마크업 언어(eXtensible Markup Language)라는 뜻의 XML은 데이터를 표현하는 형식이다.
Ajax 어플리케이션은 서버에서 클라이언트로 정보를 전달하는 데 사용하기 위한 구조화된 형식을 필요로 한다.
그러나 XML은 선택할 수 있는 여러 가지 방법 중 하나일 뿐이다,
Ajax를 사용하기 위해 구조화된 데이터를 만드는 데는 다른 방법도 있다.

- 유효한 마크업 코드

XML을 사용하지 않고도 Ajax 어플리케이션을 만들 수 있다.
그러나 Ajax의 사용여부와는 관계 없이 모든 웹사이트의 기초가 되는 또 다른 마크업 언어가 있다.

HTML(HyperText Markup Language)는 월드와이드웹의 공용어이다.
HTML은 웹에서 내용을 의미 있는 구조로 만드는 데 사용된다.
마크업 언어는 웹 페이지를 만드는데 내용 다음으로 중요하고 유용한 도구이다.

주로 Ajax가 동작하게 되는 클라이언트 쪽에서 문서를 처리하고자 한다면 적법한 마크업은 필수요소이다.
문서가 적법하지 않다면 문서를 처리하는 작업은 과도하게 복잡해지고 예측할 수 없게 된다.

마크업 요소가 올바르게 구성되어 있을 때 마크업이 적법하다고 한다.
태그는 열린 순서와 반대 순서로 닫혀야 된다.

마크업 문서가 W3C에서 지정한 가이드라인에 올바르게 맞춰져 있다면 유효하다고 간주한다.
문서에 문서 타입 선언이라고 하는 doctype을 문서의 최상단에 지정함으로써 어떠한 가이드라인을 따르고 있는지 명확하게 지정할 수 있다.

유효성을 검사하는 데 사용하는 명세가 몇 가지 있다.
이 명세들은 두 가지 다른 형태로 정리된다.
HTML과 확장성 하이퍼 텍스트 마크업 언어라고 하는 XHTML이 바로 그것이다.

HTML을 사용하면 코드를 더 느슨한 규칙에 맞춰 작성하게 된다.
일부 태그는 닫는 태그를 적을 필요가 없으며 대, 소문자를 구별할 필요도 없다.

XHTML은 XML로 재작성된 HTML이다.
XHTML이 더 엄격하다.
모든 태그는 소문자로 작성되어야 하고, 모든 요소에는 닫는 태그가 반드시 필요하다.
줄바꿈이나 이미지 같이 단독으로 사용될 수 있는 태그의 경우에는 요소의 마지막에 태그를 닫는다는 의미인 슬래시(/)를 추가한다,

규칙 엄격하게 지켜야 하는 경우는 XHTML이 좋다.
모든 요소에서 닫는 태그를 사용하면 문서 처리가 더 쉬워진다.
HTML에서도 닫는 태그를 사용할 수는 있다.
그러나 doctype을 XHTML로 선언하면 유효셩 검사에서 마크업 코드의 오류를 더 잘 잡아낼 것이다.

- CSS

HTML과 XHTML은 내용이 무엇인지 정확하게 표현하는 데 사용된다.
HTML과 XHTML은 어느 부분이 문단이고 어느 부분이 헤드라인인지 알 수 있게 해준다.
그러나 내용이 어떻게 보여질 것인가에 대해 표시하지는 않는다.
이것은 CSS가 다룬다,

CSS 선택자를 사용하면 문서 내에서 스타일을 적용하고자 하는 요소를 선택할 수 있다.
요소 선택자를 사용하면 문서 내에 포함된 모든 특정 요소에 스타일을 적용할 수 있다.

대부분의 Ajax 어플리케이션은 브라우저에서 문서의 내용을 업데이트 하는 것 뿐만아니라 스타일도 변경한다,
문서의 구조나 스타일을 갱신하기 위해서 브라우저, 문서, 문서의 스타일을 처리할 수 있는 클라이언트 측 프로그래밍,
즉 자바스크립트가 필요하다.

- DOM 스크립트

DOM은 문서의 구조를 표현하는 표준이다.
DOM 스크립트라는 용어는 표준에 근거한 문서의 동작관리를 의미한다.
DOM 스크립트는 Ajax에 절대 없어서는 안 될 기능이다.
